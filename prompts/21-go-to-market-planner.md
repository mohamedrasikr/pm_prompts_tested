# Go-to-Market Planner

**Category:** Strategy
**Use when:** You're planning a feature or product launch and need a GTM plan.

## The Prompt

```
You are a product marketing manager helping a PM plan a launch.

I'll describe what we're launching. Create a GTM plan:

**Launch Tier:** [Tier 1: Major / Tier 2: Notable / Tier 3: Minor]
Determines the level of investment in launch activities.

**Launch Date:** [Date]
**Beta Date:** [Date, if applicable]

**Positioning**
- One-liner: [What it does for whom — max 15 words]
- Problem statement: [The pain this solves]
- Key differentiator: [Why this vs. alternatives]

**Target Audience**
- Primary: [Specific segment]
- Secondary: [Broader audience]
- Anti-audience: [Who this is NOT for — avoid wasted effort]

**Launch Channels**

| Channel | Action | Owner | Date | Content Needed |
|---------|--------|-------|------|---------------|
| In-app | [Announcement banner, tooltip, etc.] | | | |
| Email | [Launch email to relevant segment] | | | |
| Blog | [Launch blog post] | | | |
| Social | [LinkedIn, Twitter posts] | | | |
| Docs | [Help center, changelog] | | | |
| Sales | [Sales enablement, deck update] | | | |
| CS | [Support team briefing] | | | |

**Enablement**
- Internal FAQ: [Questions sales/CS will get and answers]
- Demo script: [How to show the feature in 2 minutes]
- Objection handling: [Common pushback and responses]

**Success Metrics**
- Adoption: [X% of target users try the feature within Y days]
- Engagement: [Z metric indicating ongoing usage]
- Revenue: [If applicable — impact on conversion, expansion, etc.]

**Rollback Plan**
- If launch goes poorly, what do we do?
- Communication plan for issues

What we're launching:
[DESCRIBE THE FEATURE OR PRODUCT]
Target users: [WHO IS THIS FOR]
Timeline: [WHEN]
```

## Tips for Better Results

- The more specific context you provide, the better the output
- Paste real data rather than describing it abstractly
- Iterate on the output — use it as a starting point, not a final answer
