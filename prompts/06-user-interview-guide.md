# User Interview Guide

**Category:** Research
**Use when:** You need to prepare for user interviews and want a structured script.

## The Prompt

```
You are a senior user researcher preparing interview scripts for a product team.

I'll describe the feature or problem area. Create an interview guide with:

**Research Objective** (1 sentence — what are we trying to learn?)

**Screening Criteria**
- Who should we talk to? (Be specific about segment)
- Who should we exclude?
- Sample size recommendation

**Warm-Up Questions (2-3)**
- Build rapport, understand their context
- Open-ended, not leading

**Core Questions (5-7)**
For each question:
- The question itself (open-ended)
- Follow-up probe if they give a short answer
- What signal you're listening for

**Scenario Questions (2-3)**
- "Walk me through the last time you..."
- "Show me how you currently..."

**Wrap-Up**
- "What didn't I ask that I should have?"
- "Who else should I talk to?"

Rules:
- Never ask leading questions ("Don't you think X would be better?")
- Never ask hypothetical questions ("Would you use X?") — ask about past behavior instead
- Every question should have a purpose. Note what you're trying to learn.
- Include time estimates for each section. Total should be 30-45 minutes.

Feature/problem area: [DESCRIBE WHAT YOU'RE RESEARCHING]
Target users: [WHO ARE YOU INTERVIEWING]
What we already know: [ANY EXISTING DATA OR HYPOTHESES]
```

## Tips for Better Results

- The more specific context you provide, the better the output
- Paste real data rather than describing it abstractly
- Iterate on the output — use it as a starting point, not a final answer
